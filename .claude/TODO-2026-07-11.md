## Decision log

- **2026-07-11 — Project founded; harness bootstrapped (pre-PR era).**
  Name: `instruments.js` (plural; "vst.js" disqualified — Steinberg trademark forbids VST blends/derivatives + mis-signaling; plural avoids Sentry's literal `instrument.js` filename collision and the observability verb reading; full survey in the approved plan). License: dual MIT/Apache-2.0. Clean-room: papers-only for copyleft (`agentic-docs/licensing.md`). Eval panel: Keunwoo + Hayoung + 3–5 community, public at v0.5. Architecture: single-worklet Rust→WASM, SoA voices, no COOP/COEP (`agentic-docs/design/2026-07-11-architecture.md`). Multi-track arrangements are the unit of performance budgeting (PRINCIPLES #4) — added at founding by Keunwoo.

- **2026-07-11 — First sound: issues #3/#5/#6-lite/#8-lite/#10 landed in one vertical slice (branch `feat/first-sound`).**
  Root cause of "nothing playable": no engine existed. Built: Rust DSP core (modal mallets ×4, EKS plucked guitar/bass, tine e-piano, GM drum kit, polyBLEP subtractive pad; voice pool w/ stealing, track buses w/ smoothed equal-power pan, master soft-clip, denormal flushing, per-instance SR) + hand-rolled WASM ABI (no bindgen) + AudioWorklet processor (main-thread compile → postMessage → sync instantiate; quantum-boundary event queue; 1 Hz stats) + implemented `createEngine()`/tracks/`play()`/`renderOffline()` + playground (keyboard, drum pads, 6-track demo arrangement, offline WAV download) + Node verification harness.
  **Verification (numbers):** 9/9 cargo tests incl. tuning at 44.1k & 48k (440 Hz ±3%), damping, stealing, pad release; demo render 21.7 s, peak 0.64, RMS −23 dB, 0 NaN, 21 voices, **29.6 µs/quantum = 1.11% of the 2.67 ms budget** (Node/wasm); sustained-retrigger bench 43.8 µs = 1.64%. WASM **18.2 KB gzipped**. Human listening: pending (Keunwoo — this is the Q1 AB-gate input, n≥5).
  **Decision (Keunwoo, mid-build): no paradigm purity** — classic subtractive/FM voices welcome where fast+beautiful wins (PRINCIPLES #5 amended; SynthPad added, now backing strings/synth/brass/voice families instead of mallet placeholders).
  Known honest gaps → Open: quantum-boundary (≤2.7 ms) event jitter, not sample-offset; iOS Safari untested; bundler matrix untested; piano family maps to e-piano placeholder.

- **2026-07-11 — Silence bug fixed + MIDI release + first full 7-persona panel review (PR #11).**
  Root cause of Keunwoo's "things don't sound": structured-cloning a WebAssembly.Module into an AudioWorklet is silently dropped to `messageerror` on Safari/Chromium-headless (found via headless step-probe after MIME/instantiation ruled out). Fix: bytes-only init + in-worklet sync compile; offline renders deliver bytes+schedule via processorOptions (port messages aren't pumped around offline render loops — the earlier handshake await was a deadlock). Never-silent guards added (onmessageerror/onprocessorerror/red status).
  Shipped same session: @instrumentsjs/midi SMF parser (tempo maps/running status/GM/CC64), engine sustain pedal, sample-accurate segment scheduling, batched events, playground MIDI drag-drop + live Web MIDI, Playwright E2E in CI.
  **Panel review round 1 (all 7 personas, PR #11)** — fixed same-day: subarray alloc + O(N²) shift on render path (Yotam); guitar 44.1k test + render-harness divergence from shipped path + both-channel stats (Keunwoo); discontinuous soft-clip knee → slope-continuous limiter + e-piano drive key-tracked down (producer); KS loop-filter phase-delay tuning compensation — strings no longer detune with velocity — + allpass conditioning (Juhan, verified by new velocity-independence test); swing/micro-timing/composed melody contours/dynamic arc in demo + vibraphone rotary tremolo (Hayoung); honest bundler claims + ~24 KB all-in number (senior-web-dev); e2e live-path escape hatch removed. Deferred → issues: MPE/continuous control (Jordan blocking), e-piano ADAA/oversampling, sympathetic resonance/pedal bloom, drum vel→timbre, steal crossfade, bundler-matrix fixtures, FFT-grade tuning eval.
  Verification post-fixes: 11/11 cargo (incl. velocity-independent tuning), render 1.22% budget, E2E PASS (no escape hatch), 0 errors.
  Customer-zero note: integration branch in music-transcription-app was externally reverted (other work in flight there) → preserved as `demos/customer-zero/` drop-in example instead; hands off that repo.
